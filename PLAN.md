### 2. Make Differentiators Obvious on the Deck Page

The Overview tab currently mirrors what Moxfield already shows (mana curve, card type breakdown, stats). Users who know Moxfield won't see the value. The differentiators are: **weakness explanations, strategy advice, and collection-aware improvements** — but they're either buried in other tabs or not rich enough.

**Changes needed:**

**Overview tab — make weaknesses actionable, not just labels:**
- Each weakness currently shows a raw string like "Low ramp count (7 — recommend 10+)". Replace with an expandable card that explains *why* this matters, *what category of cards* to look for, and *2–3 concrete examples* (rule-based, not Gemini).
- Weakness explanations should be generated by `deck_analyzer.py`, not Gemini. They should always be present and fast, not dependent on an AI call.

**Themes — add definitions:**
- Each theme badge (Tokens, Graveyard, Aggro, etc.) should have a short tooltip or expandable definition:
  - e.g. "Tokens — generates creature tokens to overwhelm opponents through wide board presence"
  - Definitions are static and rule-based. Add them to `deck_analyzer.py`.

**Overview tab — surface value above the fold:**
- Add a brief "deck verdict" summary at the top of Overview: 2–3 sentences generated by rule-based logic (commander + themes + top 2 weaknesses). Should be instant, never blank. Gemini can optionally augment this, but rule-based is the default.
- Add a small "differentiator callout" banner or section: point users toward the Strategy tab and Collection Upgrades tab with a sentence about what they'll find there.

### 3. Analysis History — Deduplication + Deck Identity

Currently, analyzing the same Moxfield deck twice creates two `analyses` rows. There's also no deck name shown — just an ID. Needs:

**Naming:**
- Store the Moxfield deck name in the `analyses` row (already available from the Moxfield API response as `name` on the deck object).
- Store the full Moxfield deck URL (reconstruct from `moxfield_id`: `https://www.moxfield.com/decks/{moxfield_id}`).
- Display deck name + link in the analysis history list on the Dashboard.

**Deduplication — same deck, same content:**
- When `POST /api/decks/analyze` is called, before running a new analysis, check if the user already has an analysis for that `deck_id`.
- If yes, compare the deck's card content hash (or `updated_at` from Moxfield, or a hash of the sorted cardlist) to what was analyzed before.
- If content is **unchanged**: return the existing analysis immediately (HTTP 200 with existing `analysis_id`). No duplicate row, no recompute.
- If content **has changed**: run a fresh analysis, save a new row (overwrite or append — TBD, but overwriting the existing row for the same user+deck is simpler and avoids unbounded history growth).

**Implementation notes:**
- Moxfield API response includes a `lastUpdatedAtUtc` field on the deck — use this as the change signal, avoid hashing 100 cards.
- Add `deck_name TEXT`, `moxfield_url TEXT`, `deck_updated_at TIMESTAMPTZ` columns to the `analyses` table (new migration `003_analyses_metadata.sql`).
- Dashboard history list: show deck name as the primary label, with a small "View on Moxfield ↗" link next to it, and the analysis date.
- Frontend: after `POST /api/decks/analyze`, if the response includes `cached: true`, navigate to the existing analysis URL without showing a "Analyzing..." spinner.

### 4. Gemini Rate Limiting Strategy — AI as Last Resort

Free tier Gemini hits 503s and rate limits regularly. Right now, every tab that calls Gemini will fail or fall back to the rule-based fallback with no indication to the user that the content could be richer. The architecture should treat Gemini as an enhancement, not a dependency.

**Strategy: Rule-based first, AI as enrichment**

**For Strategy tab:**
- Build a full rule-based strategy guide in `deck_analyzer.py` / `gemini_assistant.py` using the existing analysis data (themes, commander, weaknesses, avg CMC, ramp count, etc.)
- The rule-based output should be genuinely useful (3–5 paragraphs with real advice): game plan based on theme tags, mulligan heuristics based on avg CMC + ramp count, win condition categories based on theme, matchup notes based on removal/interaction counts
- Only call Gemini if: (a) rule-based strategy is ready and (b) user clicks a **"Enhance with AI"** button, OR Gemini is available and the response can be appended/replaced non-disruptively
- Never block on Gemini. Render the rule-based version immediately, then optionally upgrade it.

**For Improvements tab:**
- Rule-based improvements already partially exist via `find_collection_improvements()` in `deck_analyzer.py`. Wire this up to the Improvements tab as the primary content.
- Group suggestions by weakness category (e.g. "Ramp", "Draw", "Removal") with 3–5 card suggestions per category using Scryfall data where possible.
- Gemini is called only to add explanatory prose around the rule-based suggestions — again, as a non-blocking enhancement.

**For Scenarios tab:**
- The before/after analysis can be structured rule-based: recalculate `analyze_deck()` after applying the swap. Show the delta in key stats (ramp +1, avg CMC change, new weaknesses introduced, themes gained/lost).
- Gemini adds qualitative narrative — but the quantitative diff should always render instantly.

**Implementation notes:**
- Add a `GeminiStatus` indicator in the UI (subtle, e.g. small colored dot in the tab header): green = AI enhanced, yellow = AI unavailable (showing rule-based), grey = loading
- Log all Gemini 503/429 errors server-side. Consider a simple in-memory cooldown: if Gemini returned 503, skip attempting Gemini calls for the next 60 seconds for that process.
- Backend: wrap all `_ask()` calls with a retry-with-backoff (1 retry, 2s delay) before falling back. This handles transient 503s without making the user wait long.
- Do not expose Gemini errors to the user — always show useful content. The error message should only communicate "AI advice not available right now" if they explicitly requested AI enhancement.

## Goals

1. Users can get recommendations on how to alter their decks based on their card collection (data from Moxfield)
2. Users can get recommended play strategies for their current decks
3. Users are shown before/after scenarios of how their play strategy changes based on suggested deck changes
4. Multi-user support (~5 users), accessible on phone and desktop

## Architecture Decisions

| Concern | Decision | Notes |
|---|---|---|
| Frontend | React (Vite) + TailwindCSS | Deployed on Vercel (free) |
| Backend | FastAPI (Python) | Deployed on Render (free tier, sleeps after 15 min idle — acceptable for ~5 users) |
| Database + Auth | Supabase | PostgreSQL + Google OAuth + admin allow-list |
| AI | Gemini 2.5 Flash | Free tier, 1M token context window |
| Moxfield decks | Unofficial public API | `api2.moxfield.com/v2/decks/all/{id}` — public decks only, no auth needed |
| Moxfield collections | CSV upload only | No public OAuth exists for private collection data; Moxfield supports CSV export natively |
| Auth model | Google login + allow-list | Admin adds approved emails to `allowed_users` table in Supabase |

## MTG Commander Expertise Reference

> Reference section for testing and developing the rule-based engine. All thresholds, staple lists, and detection patterns should be validated against this section. Updated as we learn from real deck testing.

### Commander Format Fundamentals

- 100-card singleton, 1 commander (sometimes 2 with Partner)
- 3 opponents in a typical game — threats come from all directions
- Games run 8-15 turns on average; early ramp is critical
- Color identity restriction: every card must fit within the commander's color identity
- Higher variance than 60-card formats — consistency tools (draw, tutors) are more important

### Community-Consensus Thresholds

| Category | Minimum | Good | Excellent | Notes |
|---|---|---|---|---|
| Lands | 36 | 37 | 38 | Below 33 is dangerous; above 42 floods |
| Ramp (CMC ≤ 3) | 10 | 12 | 15 | Only count acceleration, not 5+ CMC rocks |
| Card draw / advantage | 10 | 12 | 15 | Repeatable engines > one-shot effects |
| Single-target removal | 8 | 10 | 12 | Must include mix of creature + noncreature answers |
| Board wipes | 2 | 3 | 4 | Critical for recovering from falling behind |
| Avg CMC (nonland) | — | ≤ 3.0 | ≤ 2.5 | Above 3.5 is slow; needs extra ramp to compensate |

### Oracle Text Detection Patterns

**Ramp (mana acceleration) — must cost ≤ 3 CMC to count as ramp:**
- `"add {"` — Sol Ring, Llanowar Elves, most mana rocks
- `"add one mana"` / `"add mana"` — Arcane Signet, Birds of Paradise, Bloom Tender
- `"add {c}"` variations — Mind Stone, Fellwar Stone
- `"treasure token"` / `"create a treasure"` — Dockside Extortionist, Smothering Tithe, Pitiless Plunderer
- `"search your library"` + `"land"` (on instants/sorceries) — Cultivate, Kodama's Reach, Nature's Lore
- `"mana dork"` heuristic: creature + (`"add {"` or `"add one mana"`) + CMC ≤ 2
- **Exclude:** cards costing 4+ CMC are "mana sources" not "ramp" (Gilded Lotus, Thran Dynamo)

**Card draw:**
- `"draw a card"` — universal, catches most
- `"draw cards"` / `"draw two"` / `"draw three"` / `"draw x"` — bulk draw
- `"draw an additional card"` — Phyrexian Arena, Sylvan Library
- `"draw that many"` — variable draw
- `"put that card into your hand"` — Necropotence, Bolas's Citadel-style effects
- `"look at the top"` + `"put into your hand"` — Impulse, Anticipate (minor card selection)

**Single-target removal:**
- `"destroy target"` — Go for the Throat, Beast Within, Generous Gift
- `"exile target"` — Swords to Plowshares, Path to Exile, Reality Shift
- `"deals damage to any target"` / `"deals damage to target creature"` — Bolt effects
- `"becomes a"` (enchantment-based neutralization) — Darksteel Mutation, Kenrith's Transformation, Imprisoned in the Moon
- `"shuffle"` + `"into"` + `"library"` — Chaos Warp, Oblation (tuck effects)
- **Do NOT count:** `"return target"` (bounce) — bounce is not real removal in Commander; opponents replay the card

**Board wipes:**
- `"destroy all creatures"` — Wrath of God, Damnation, Supreme Verdict
- `"destroy all nonland"` / `"destroy all permanents"` — Austere Command, Planar Cleansing
- `"exile all"` — Farewell, Merciless Eviction
- `"damage to each creature"` / `"deals X damage to each creature"` — Blasphemous Act, Chain Reaction
- `"-X/-X"` + broad target (creatures, all, each) — Toxic Deluge, Black Sun's Zenith
- `"return all"` + `"to their owners' hands"` — Cyclonic Rift (overloaded), Evacuation

**Interaction (instants + sorceries that interact):**
- Everything in removal above (on instant/sorcery type lines)
- `"counter target spell"` — Counterspell, Swan Song, Arcane Denial
- `"counter target"` (broader) — catches Negate, Dovin's Veto, etc.

### Theme Detection — Card-Level Counting

> Count individual **cards** that match, not keyword hits in a text blob. A single card counts once per theme max. Minimum card thresholds prevent false positives.

| Theme | What to count | Min cards | False positive traps |
|---|---|---|---|
| **Tokens** | Cards that create creature tokens + anthem effects (`"creatures you control get +"`) | 8 | Treasure/Food/Clue tokens do NOT count — only creature tokens. Check for `"creature token"` specifically. |
| **Aristocrats** | Sacrifice outlets (`"sacrifice a creature"`, `"sacrifice a permanent"`) + death triggers (`"when this creature dies"`, `"whenever a creature dies"`) + drain effects (`"each opponent loses"` + life context) | 8 | Fetch lands and `Sakura-Tribe Elder` have "sacrifice" — don't count lands as Aristocrats cards. Require creature/enchantment/artifact type. |
| **Graveyard** | Cards interacting FROM graveyard (`"from your graveyard"`, `"flashback"`, `"unearth"`, `"escape"`, `"embalm"`, `"encore"`) + self-mill (`"mill"`, `"put into your graveyard from your library"`) | 7 | Generic "graveyard" mention alone (e.g. Bojuka Bog as graveyard hate) should not count — require FROM graveyard or filling it. |
| **Voltron** | Equipment cards + Auras that target creatures (`"enchant creature"`) + cards that buff the equipped/enchanted creature (`"equipped creature gets"`, `"enchanted creature gets"`) | 8 | A deck with 2 Lightning Greaves and 1 Swiftfoot Boots is NOT Voltron — those are generic protection staples. |
| **Spellslinger** | Payoffs (`"whenever you cast an instant or sorcery"`, `"magecraft"`) + instants/sorceries making up 30%+ of nonland cards | 3 payoffs + 30% ratio | Pure control decks run lots of instants but aren't spellslinger — require the payoff cards. |
| **Tribal** | Creatures sharing a type at 60%+ of all creatures + lord effects (`"other [type] you control"`, `"[type] creatures you control get"`) | 60% + 2 lords/payoffs | Generic "creature" doesn't count — must share a specific subtype. |
| **Landfall** | Cards with literal `"landfall"` keyword or `"whenever a land enters the battlefield under your control"` | 6 | Don't count lands themselves. |
| **+1/+1 Counters** | `"put a +1/+1 counter"`, `"+1/+1 counter on it"`, `"proliferate"`, `"with X +1/+1 counters"`, `"modular"`, `"evolve"` | 8 | Cards that enter with a single +1/+1 counter (e.g. random creatures with `"enters with a +1/+1 counter"`) are only mildly thematic — weight payoffs higher. |
| **Enchantress** | `"whenever you cast an enchantment"` payoffs + enchantments at 25%+ of nonland cards | 2 payoffs + 25% ratio | Don't tag just because Rhystic Study is an enchantment. |
| **Artifacts matter** | `"whenever you cast an artifact"` / `"whenever an artifact enters"` payoffs + artifacts at 30%+ of nonland cards | 2 payoffs + 30% ratio | Mana rocks alone don't make a deck Artifacts-matter. |
| **Control** | Counterspells (5+) + board wipes (3+) + low creature count (<15 nonland creatures) | Composite | A creature-heavy deck with a few counters is not Control. |
| **Aggro** | Avg CMC ≤ 2.8 + creatures at 28+ + evasion/haste keywords on 10+ creatures | Composite | Fast decks that are combo, not aggro, may have low CMC — check creature density. |
| **Combo** | **AI only — cannot be reliably detected rule-based.** Combo requires knowing which specific 2-3 card interactions produce infinite loops. | N/A | Never guess combo from keywords like `"untap all"`. Flag for AI. |

### Rule-Based vs AI Content Split

| Content | Source | Why |
|---|---|---|
| Resource counts (ramp/draw/removal/wipes) | Rule-based | Must be accurate — these are the core trust metrics |
