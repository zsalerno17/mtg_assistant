Implementation Checklist — Mobile (<768px)

  • Screen size: This spec applies to <768px only. At ≥768px, switch to Direction A (Data Dashboard) desktop spec.
  • Commander hero: MASSIVE vertical layout, 280×392px art centered, full-width gradient section, 48px vertical padding
  • Hierarchy: Commander dominates viewport → Weaknesses second section (prominent red callout) → Stats horizontal scroll → Charts vertical flow
  • Weaknesses: Second major section after hero, prominent red tinted background, single-column on small mobile (2-col on ≥600px)
  • Stats: Horizontal scroll row, 40px stat values (mobile-optimized), min-width 140px cards
  • Charts: Single-column vertical flow (natural for mobile), 200px height (240px on ≥400px)
  • Typography: Cinzel ONLY for commander name (40px mobile, 48px on ≥400px). Inter everywhere else.
  • Touch targets: Buttons 48px height (primary), 44px height (secondary), full-width on mobile, side-by-side on ≥600px
  • Spacing: 64px section spacing, 24px card padding, 16px container padding (mobile safe area)
  • Colors: Approved palette tokens (--accent-primary blue, --accent-danger red, --surface, --border)
  • Accessibility: Focus rings, semantic HTML, proper touch target sizes (min 44×44px)
