Implementation Notes

About These Mockups

These charts are rendered with D3.js to demonstrate visual design direction only. Production implementation will use Recharts, which is already integrated in the project and provides the same visual capabilities with cleaner, more maintainable React component code.

All design elements shown here (gradients, colors, font sizes, animations) can be replicated in Recharts using the same SVG techniques.

Why Recharts for Production?

Aspect Recharts Pure D3
Lines of Code (per chart) ~20-30 lines (JSX components) ~80-120 lines (imperative SVG)
Learning Curve Easy (React-like declarative) Steep (D3 selections, scales)
Maintainability High (component-based) Medium (imperative cleanup)
Visual Quality Excellent (same SVG capabilities) Excellent
Already Integrated Yes (v2.8.0 installed) No (would need to add)

Recommendation: Which Designs to Implement?

Recommended selections:

  • Pie Chart: Variation A (Donut + Gradients) — best balance of visual interest and clarity
  • Mana Curve: Variation A (Area Chart) — smooth, modern, engaging

Key improvements to implement in DeckPage.jsx:

  • Remove all labels from PieChart (use grid view as legend)
  • Apply smart MTG color mapping (Creatures→Green, Instants→Blue, etc.)
  • Add radial gradients to pie slices via SVG defs
  • Replace mana curve BarChart with AreaChart (smooth curves)
  • Reduce font sizes to 10-11px (design system scale)
  • Remove Y-axis from mana curve (cleaner look)
  • Increase padding angle between pie slices (4 instead of 2)
  • Add hover states that highlight corresponding grid items
